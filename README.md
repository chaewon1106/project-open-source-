# Simple Profile Card Generator

Simple Profile Card Generator는 사용자가 입력한 정보를 바탕으로 담백하고 깔끔한 디자인의 프로필 카드를 생성해주는 웹 애플리케이션입니다. 사용자는 다양한 색상 테마와 문체를 선택할 수 있으며, 생성된 카드는 고해상도 이미지로 저장하거나 텍스트로 복사 및 공유할 수 있습니다.

## 프로젝트 개요

* **프로젝트명**: Simple Profile Card Generator
* **목표**: 복잡한 디자인 툴 없이 누구나 쉽게 자신만의 개성이 담긴 소개 카드를 제작할 수 있도록 지원
* **주요 기능**: 프로필 정보 입력, 실시간 테마 적용, 문체별 소개글 자동 생성, 이미지 저장 및 공유

## 기술 스택 (Tech Stack)

이 프로젝트는 외부 프레임워크 의존성을 최소화하여 Vanilla JavaScript로 구현되었습니다.

### Core
* **HTML5 / CSS3**: 시맨틱 마크업 및 CSS Variables를 활용한 동적 테마 시스템
* **JavaScript (ES6+)**: DOM 조작, FileReader API, 비동기 데이터 처리

### Libraries & APIs
* **html2canvas**: DOM 요소를 캔버스로 변환하여 이미지 다운로드 기능 구현
* **Web Share API**: 모바일 환경에서의 네이티브 공유 시트 연동
* **Clipboard API**: 생성된 텍스트의 원클릭 복사 기능
* **Pretendard**: 가독성과 심미성을 고려한 웹 폰트 적용 (CDN)

## 페이지 구성 (Page Composition)

애플리케이션은 사용자 흐름에 따라 3단계의 화면(View)으로 구성되어 있습니다.

**1. Intro View**
* CSS Animation을 활용한 입체적인 카드 그래픽으로 시각적 흥미를 유도합니다.
* 시작 버튼을 클릭하면 폼 입력 화면으로 부드럽게 전환됩니다.

**2. Form View**
* **데이터 입력**: 이름, 프로필 사진, MBTI, 음식, 취미 등 사용자 정보를 입력받습니다.
* **테마 선택**: 5가지 컬러 테마(Ocean, Peach, Mint, Indigo, Gray)를 제공하며 선택 시 즉시 반영됩니다.
* **문체 선택**: 비즈니스, 일상, 정보 중심, 감성 등 상황에 맞는 문체를 선택할 수 있습니다.
* **편의 기능**: 입력이 고민되는 사용자를 위해 MBTI 및 취향 랜덤 선택 버튼을 제공합니다.

**3. Result View**
* 입력된 정보와 테마가 적용된 최종 프로필 카드를 렌더링합니다.
* 선택한 옵션에 따라 자연스러운 자기소개 문구가 자동으로 생성됩니다.
* 이미지 저장, 텍스트 복사, 공유하기 기능을 통해 결과물을 활용할 수 있습니다.

## 개발 이력 (GitHub Activities)

본 프로젝트는 다음과 같은 개발 흐름으로 진행되었습니다.

1.  **초기 구조 설정 (Skeleton)**
    * HTML 기본 뼈대 구축 및 프로젝트 구조 설정

2.  **레이아웃 및 항목 수정**
    * 기획 단계에서 누락된 입력 항목 추가
    * 전체적인 UI 레이아웃 및 요소 위치 재조정

3.  **시각적 요소 개선**
    * 색상 팔레트 구체화 및 썸네일 스타일 수정
    * 디자인 일관성 확보

4.  **핵심 로직 구현**
    * MBTI 데이터 처리 로직 구현
    * 사용자 업로드 이미지를 결과 카드 아바타에 반영하는 FileReader 기능 구현
    * 선택한 옵션에 따른 간단한 소개글 자동 생성 로직 개발

5.  **기능 통합 및 스타일 고도화**
    * html2canvas 라이브러리를 연동하여 카드 이미지 저장 기능 구현
    * 텍스트 복사 버튼 활성화
    * 색상 및 테마 적용 안정화 작업
    * 마이크로 인터랙션(애니메이션) 추가 및 최종 CSS 스타일링 개선

6.  **UX 강화 및 페이지 확장**
    * 앱의 진입점인 인트로 페이지 추가
    * 기존의 경고창(Alert)을 사용자 친화적인 토스트 알림(Toast Notification)으로 대체
    * Web Share API를 활용한 공유하기 기능 탑재로 UX 최종 강화

## 회고 (Retrospective)

프로젝트 개발 과정에서의 기술적 도전과 배운 점입니다.

**고해상도 이미지 캡처 최적화**
html2canvas 라이브러리 사용 시 기본 설정으로는 결과 이미지가 흐릿하게 렌더링되는 문제가 있었습니다. 이를 해결하기 위해 캔버스의 scale 옵션을 조정하여 고해상도(Retina 디스플레이 대응) 다운로드를 구현하였으며, CORS 설정을 통해 외부 리소스 로딩 문제를 방지했습니다.

**동적 텍스트 생성 시스템**
단순한 정보의 나열이 아닌, 사용자가 선택한 '문체'와 '연락 선호 시간'에 따라 문장의 어미와 조사가 자연스럽게 바뀌는 템플릿 리터럴 로직을 구현했습니다. 이를 통해 사용자에게 맞춤형 콘텐츠 경험을 제공할 수 있었습니다.

**사용자 경험(UX) 중심의 기능 개선**
기능 구현에만 그치지 않고 사용자의 실제 경험을 고려했습니다. 브라우저 기본 Alert 창 대신 직접 구현한 토스트 메시지를 사용하여 시각적 방해를 줄였으며, 모바일 환경에서의 사용성을 고려해 네이티브 공유 기능을 추가함으로써 완성도를 높였습니다.
